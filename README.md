# xyz_2st_project

자동주유 + 멀티모델을 이용한 프로젝트

시나리오 구성

실제 사용자 대화 시나리오 (실사용 예시)
상황: 서울의 무인 주유소에 도착한 운전자 (김철수)
차량: 현대 아이오닉 5 (전기차 + 수소차 호환 가능)
도착 시간: 2025년 11월 10일 오후 3:44 (KST)
위치: 서울 강남구, 자동 주유 레인

1. 도착 & 음성 인사 + 차량 자동 인식
AI (스피커/앱 알림):
“안녕하세요, 김철수님. 번호판 12가 3456 확인되었습니다.
오늘은 수소 충전을 원하시나요, 아니면 급속 충전을 원하시나요?”
김철수 (음성):
“수소로 80%까지 채워줘.”
→ 멀티모달 입력 처리
비전: 번호판 + 차량 모델 인식 (YOLO + OCR)
음성: 자연어 이해 (Whisper + LLM)
센서: 수소 탱크 잔량 자동 감지 (IoT 센서)
사용자 DB: 과거 충전 기록 (텍스트 로그)

3. 충전 중 대화 (부가 서비스 제안)
AI:
“현재 42% → 80%까지 약 18분 소요 예정입니다.
커피 한 잔 하시겠어요? GS25 로봇 배달로 3분 내 도착 가능합니다.”
김철수:
“아이스 아메리카노로 부탁해.”
→ 멀티모달 연계
결제: NFC + 음성 인증
외부 API: GS25 로봇 배달 시스템 연동
위치: 사용자 GPS + 주유소 내부 카메라로 배달 위치 추적

5. 완료 & 요약 대화
AI:
“충전 완료! 42% → 80% (38% 충전, 4.2kg)
총 금액: 42,000원 (카드 자동 결제)
다음 충전 예상: 11월 15일 오전
아이스 아메리카노도 도착했습니다. 즐거운 운전 되세요!”
김철수:
“고마워, 편하네.”
핵심: 어떤 멀티모달 기술이 대화에 쓰였나?
